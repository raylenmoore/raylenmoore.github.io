关于请他人帮助的一些趣事

发表于 2018-12-07

很多人都不好意思请他人帮忙，哪怕是当他真正需要他人的帮助的时候也难以启齿，不是担心向他人求助，就是怕遭人拒绝。我们来听听一个故事可能会改变你的这种想法。

本杰明.富兰克林原本是费城一个默默无闻的小人物，当时他想得到某个重要人物的垂青，于是经常设法请那个人帮他一些无关紧要的小忙，比如说借一本书给他之类的。您可能要问，这样做有什么用，您还别说，富兰克林的这些做法产生了一个对他前途极有帮助的结果：后来，那个大人物更加欣赏和信任富兰克林。其实这里蕴含了一个有趣的心理学问题，叫做“避免不一致性倾向”。分析一下，一开始富兰克林找那个大人物借一些书啥的，这些事对大人物来讲简直是举手之劳，到后来富兰克林再找他帮其他更大的忙，这个大人物就会有意无意地会维持之前乐于助人的形象，从而继续帮富兰克林一些难度可能远超过借一本书之类的小忙了。

克里斯.马修斯在他的著作《赢球 政治是这样玩的》有这样一番见解：
擅长利用别人的长处，是一个更重要的技巧，索取比给与更好，你需要懂得向人求助。当一个人自身还不够强大的时候，懂得求助时一门神技。人们总觉得向人求助时示弱的表现，不好意思开口，其实并不是这样，向人求助提供了一个让对方展现价值的机会，而对方一旦开始帮助你，往往就停不下来了，一般都会一帮到底。就像马基雅维利说的，伟大的事业就像伟大的战争，一旦做出了牺牲，就需要进一步做出牺牲。别人之所以会一而再，再而三对你伸出援手，这里边还涉及另一个概年念，叫沉没成本，因为他之前已经付出一些，如果不继续帮你，之前的付出有可能付诸东流。

不过最后必须要补充一点，自己要争气，让人值得帮你，这是关键。

《这个世界土崩瓦解了》读后感

发表于 2018-09-04 | 更新于 2018-12-07

故事梗概

《这个世界土崩瓦解了》主要讲述一个非洲氏族在欧洲殖民主义的侵略下土崩瓦解的故事。故事的主人公奥贡喀沃(以下简称奥)出生在一个并不富裕的家庭，但是他靠自己勤劳的双手，以及对荣誉和财富的追求，终于获得氏族中一些受人尊敬的头衔和殷实的家财。故事的转折发生在奥因误杀同族人而逃亡到他母亲所在村庄的7年中：欧洲殖民者来了，他们在村庄里新建教堂，学校，医院，还有新政府。等奥流亡归来后，这些殖民者已站稳脚跟。他试图武力驱逐这些白人，使村庄回到以前的状态，然而最终以失败告终，奥本人最后上吊自杀。

Q & A

为什么基督教会在村庄中逐步发展壮大？

要回答这个问题，可以从村庄基督教堂的早期成员获得一些线索：
第一类成员是对他们本族的宗教信仰产生困惑甚至失望的人，最典型是奥的儿子恩沃依埃。他曾有过两次内心的弦崩断的经历，第一次是听到双胞胎婴儿（在村中双胞胎被视为异物）被人遗弃的哭声，另一次是得知伊克美弗纳（来自另一个村庄，在恩沃依埃家住了3年且叫奥贡喀沃父亲）被奥亲手砍死的事。这些事令恩沃依埃感到痛心而又困惑不解，而适时到来的基督教对恩沃依埃来说有如“久旱逢甘霖”，在他最后一次和他父亲发生激烈冲突后便毅然决然地投入新宗教的怀抱 。
第二类成员是在氏族村庄中地位低下的贱民、乞丐或其他一些混地不好的人。很其它村民相比，这些人无论是物质生活还是精神生活都低人一等。
这两类人是新宗教的忠实拥孬。

奥拥有自己的枪支，但是在后来跟白人的交锋中为什么不用枪，而是用砍刀？

书中描写奥的两次用枪，第一次是奥在愤怒中向她妻子开了一枪（结果没打中），第二次是奥在参加同村人葬礼时枪走火误杀村民。从奥的这两次用枪不难看出，奥并不擅长用枪，也有可能他手中的枪本身质量也好不到哪里。而刀则不同，“他在战场上杀过五个人”，刀为他带来荣誉，是奥信赖的武器，他尚未意识到枪的巨大威力。

奥为什么要上吊自杀？

不得不说，奥上吊自杀这个结局是完全出乎我意料之外的。前文说奥是一个勇敢的战士，即使整个村庄都反对战斗，他也要为自己报仇雪恨——完全看不出任何上吊自杀的端倪。我唯一能想到的原因是，奥最终意识到敌我力量的悬殊，而他不想再连累自己的家人和朋友。

如果在白人来临是奥不是流亡在外，而是在自己村庄，故事会不会有不同的结果？

迷信落后的氏族在西方精良的火器，仁慈宽容的新宗教，现代医学与科技的进攻下土崩瓦解是是历史的必然，是大势所趋。假设白人势力在尚未站稳脚跟之前就遭到奥等人发动的进攻，那也只是暂时延缓白人势力的发展壮大，多死几个白人而已。

如果我是奥能让那个氏族社会保持原样吗？如何做？

如果自己和奥一样，是土生土长的伊博人，那我多半也跟他一样，会试图武力驱逐外来势力。因为自己付出毕生努力，眼看就要实现的人生目标就要毁于一旦，换做谁都不会甘心的。

如果自己是穿越到那个时代，并且带着现代人的知识，那情况自然有所不同。第一个思路还是和奥一样，维护旧社会，但考虑到敌人武器的精良，不能直接对抗，只守不攻。这又有几种可能，首先寻找易守难攻可做屏障的天然地理，可惜他们似乎没有，那就只能构建足以抵挡殖民者的入侵的防御工事，建立一个坚不可摧的围墙，把氏族包围起来，需外界断绝往来。但这不是一朝一夕能完成的，即使成为现实，这样自我封闭的村庄相比也难以持久。

另一条路是合作，但这需要相当高外交手腕，搞不好变成压迫本族人的傀儡，搞得好多少能给本族人带来一些福祉。这条路似乎是更值得一试。

本书价值

  1. 作者在书中记录了非洲氏族社会的风俗习惯，宗教信仰，人们生活方式，寓言故事，神话传说，格言习语。在欧洲殖民文化和现代科技的强烈冲击下，如今这些内容或许只有当地少数还活着的老人依稀有印象。所以，这是作者对世界文化的贡献！

  2. 作者成功塑造了奥贡喀沃这个人物形象。当外部世界发生翻天覆地的变化时，奥贡喀沃没有考虑如何适应新环境，而是试图通过推翻新势力以便维护自己的旧势力。而他所采用的方法又是只有在就世界才行得通那套，因此，他的悲剧是不可避免的。

《叔本华：怎样读书才有效》读后感

发表于 2018-08-09 | 更新于 2018-12-07

叔本华在《怎样读书才有效》一文写过这样一段话：

人生是痛苦的，造成痛苦的根源在于意志而不在于理智，意志的本质就是盲目的欲望和不可遏止的冲动。当欲望不被满足，人会感到痛苦；当欲望得到了满足，目标即刻丧失，人便陷入无聊之境。因此，人生就像钟摆，总是在痛苦和无聊之间来回摆荡。

我认为叔本华的这番话是有一定道理的，但也有些片面。

我们来分析一下这段话。首先叔本华一上来就说“人生是痛苦的”，原因是人总有欲望不被满足——作为一名悲观主义者，说出这样的话实在没什么好奇怪的。那么，如何解除这种痛苦呢，我想唯一的办法，就是设法让欲望得到满足。但叔本华又说，那会带来另一个问题：让人陷入无聊之境。满足欲望后一定会无聊吗？为了回答这个问题，我回顾了自己以前的经历，得到的答案是肯定的。即使满足欲望后不会马上无聊，但因其带来的幸福与快乐也不会持续太久。有研究发现，很少有经历能对我们产生超过3个月的影响。当好事来临时，我们会庆祝一阵，然后冷静下来。有诗为证，苏轼的《庐山烟雨》：

庐山烟雨浙江潮，未到千般恨不消。
到得还来别无事，庐山烟雨浙江潮。

看来，欲望是个坏东西，不是让人痛苦，就是让人无聊，这在古今中外似乎已达成共识。然而，叔本华的话虽没错，但不完整。

一来，他过于强调结果，忽视过程。事实上，单从时间跨度来讲，过程远比结果更重要，人生这座钟摆，离开一端的痛苦抵达另一端的无聊之前，中间势必要有一段波澜起伏的经历。

二来，他过于过于强调痛苦，而忽视其他感情。首先，当一个人追求某种东西时，除了感到痛苦，同时也会有成就感，只要他坚信自己的所做所为是有意义的。当他不断地取得进步，离目标愈来愈近，还有持续不断地加深其满足感。其次，假设一个人从事工作碰巧是自己的兴趣所在，那更要另当别论，这种人做事情可能连目标都没有，抑或有目标但并不关心是否真正实现，因为他在做这件事本身就给他带来莫大的快乐，不但不会无聊，更不会有痛苦。

我记得大学时有位哲学老师说，叔本华虽然是对的，但不建议像他那样想，否则生活会变得很无趣。

恰如身体只能吸收同性质的东西，任何读书人也只能记住他感兴趣的东西，也就是与其思想体系或目的相合之物。任何人当然都有自己的目的。然而，类似思想体系这样的东西绝非任何人都具备。缺乏独特的思想体系之人无论对什么都不会产生客观的兴趣，因此，他们读书必定是徒劳无功，毫无心得。

怎么理解叔本华这段话？既然任何人只能吸收与其思想体系或目的相合之物，那么反过来说，一个人，必须要先建立思想体系或抱持某种目的，读书才能有所心得，否则就会徒劳无功。叔本华在这里实际上告诉我们两条学有所成的路径，一是先建立自己独特的思想体系，然后不断地往上添砖加瓦。然而正如叔本华自己所指出的“类似思想体系这样的东西绝非任何人都具备”，这条路似乎只适合那些已经学有所成的走。幸好，还有第二条路，抱持某种目的。这似乎是不言自明的—任何人读书都有自己的目的。然而事实并非如此，很多人（包括我自己在内）并不清楚自己读书的目的，至少有时不那么明确。以下是有关学习的几种常见误区：

  • 盲目听从某些所谓专家或大牛建议，却浑然自己当前的水平；当然不是说他们推荐的书不好，我这里想表达的是即使一个领域的经典著作，也不是人人都适合阅读。
  • 不筛选可用的学习资源，来者不拒。结果造成，所学的并不是自己想要的，或者重复学自己已经掌握的资料。
  • 没有明确的学习目标，没有计划也没有回顾，浑浑噩噩。
  • 误以为知识越多越好。从某种角度讲，知识确实越多越好，但考虑到人的记忆力和注意力有限，如果经过自己大脑的思考消化，什么知识都学，最终只会造成“劣币驱逐良币”。

Emacs快捷键定制方法详解

发表于 2018-07-01 | 更新于 2018-12-07

有人把Emacs比做一辆组装车，言下之意它可以按用户的需求定制方方面面，其中之一便是它的快捷键。定制一套简单易记的快捷键，可以极大地发挥出Emacs的优势。本文将完成以下这些常用快捷键的定制：

  1. Ctrl-c（复制），Ctrl-v（粘贴），还有Ctrl-x（剪切），Ctrl-z（撤销）
  2. Ctrl-s（保存）
  3. Ctrl-f（查找），左右方向键查找上一个、下一个
  4. Ctrl-Tab（切换窗口）

作为一名Windows系统编辑器的重度用户，我对一些快捷键已根深蒂固，如Ctrl-c是复制，Ctrl-v是粘贴，乍一接触Emacs由于这些熟悉的快捷键集体罢工，导致用Emacs编辑文档效率不升反降，因为所有操作都要用鼠标点击工具栏上按钮才能完成。所以，当务之急，先从定制复制、粘贴这些最常用的操作开始。事实上，这几个键的定制方法很简单：

接下来试着对文件做些修改，按Ctrl-s保存。咦，怎么没效果，编辑器底部(Emacs称之为minibuffer)怎么多出一个I-search的输入框。原来，Ctrl-s被绑定I-search命令（增量查找）

好，接下来我开始介绍如何通过调用global-set-key来绑定Ctrl-s到保存命令的方法。

global-set-key顾名思义，就是设置全局快捷键了。它的语法是：

(global-set-key key 'command)

global-set-key函数有两个参数，第一个参数key表示我们要使用的快捷键，这里我们借助kbd宏表示Ctrl-s如下：

(kbd "C-s")

更多关于键的表达法，参考KeyboardMacros

第二个参数command表示要执行的命令。看到这里，读者可能要问，我怎么知道保存对应的命令是什么呢。这就要用到Emacs提供的帮助功能了。方法如下：

  1. 按C-h调出help命令，这时底下窗口会提示Type ? for further options
  2. 按k，这时提示变为Describe the following key, mouse click, or menu item:
  3. 用鼠标点击工具栏的保存按钮

这时会新弹出一个窗口显示如下内容：

<tool-bar> <save-buffer> runs the command save-buffer (found in
global-map), which is an interactive compiled Lisp function in
‘files.el’.

It is bound to C-x C-s, <menu-bar> <file> <save-buffer>.

由此可见，保存命令是save-buffer，它已经被绑定到C-x C-s快捷键序列。

顺便说一句，Emacs的帮助系统功能很强大，它号称是一款自文档编辑器（self-documenting editor）并非浪得虚名。我们再按照类似的方法，执行以下命令看得到什么结果：
C-h k，再按C-s，输出：

C-s runs the command isearch-forward (found in global-map)

由此验证了我们之前观察到现象——Ctrl-s被绑定到查找命令（isearch-forward，这个命令我们暂且记下，下一步要用）。

我们再回到保存的这个问题上(对于使用频率很高的操作，要按两个快捷键才能完成还是显得效率低下）。既然我们已经知道快捷键的表达方式，也知道了保存的命令，我们就可以重新绑定了，在Emacs初始化文件中加以下代码：

(global-set-key (kbd "C-s") 'save-buffer)

保存文件，重启Emacs使改动生效。验证一下，按Ctrl-s可以保存了，I-search输入框也不再弹出。

使用Emacs和坚果云构建自己的笔记系统

发表于 2018-06-22 | 更新于 2018-12-07

截止目前，我主要使用两款笔记，印象笔记和为知笔记，但两款笔记都有缺陷，前者不支持多层目录结构（顶多是在笔记本外再套一层笔记本组），后者的性能实在不敢恭维（新建笔记时有明显的卡顿，有时搜索要等待半天才返回结果）。于是乎，我又萌生了更换笔记系统的念头，这次我决定自己动手搭建一个。这个笔记系统有以下优点：

  • 透明 每条笔记就是磁盘上的一个文件，每个笔记本就是一个文件夹
  • 安全 每个终端设备(家里电脑，公司电脑，手机)都备份一套数据，当然坚果云服务器上还有一份，这样即使某个设备丢失或损坏，也很容易找回所需数据。另外坚果云还保存了一定时间段内的历史版本（免费和收费版本有差别，具体可查看他们官网）
  • 不依赖于某一特定的服务 本文虽然采用坚果云，但是如有必要，读者可随意换成其它更合适的文件同步服务。

在正式开始搭建我们的笔记系统前先看一张效果图。

坚果云

  1. 我们先到坚果云注册个账号，下载并安装其PC客户端。客户端安装完成后，检查一下设置，建议勾选“开机自动启动”。

  2. 创建同步文件夹
    新建一个文件夹km（意为Knowledge Management），用于存放所有需要同步的文件。打开坚果云客户端，点击创建同步文件夹，按提示将km设置为同步文件夹。

注意：km所在路径不要包含中文，空格等特殊字符，以免有些Emacs插件无法正常工作。以下均假设km路径为C:/km，不再重复说明。

遗留问题

这个笔记系统还有些问题善待解决：

  1. treemacs的Project Root无法配置成相对路径
    当我们在一台电脑上配置treemacs的Project root时，treemacs会缓存相关信息到emacs_home\.emacs.d\.cache\treemacs-persist中，但保存的是绝对路径（即使我们设置的时候使用相对路径也会自动被转化成绝对路径），这样在其它电脑上使用treemacs时便会找不到相关目录。我曾经尝试强制把treemacs-persist中的路径改成~/..，但是发现treemacs的刷新功能会报错。
    临时解决办法：
    不让坚果云同步emacs_home\.emacs.d\.cache文件夹，具体操作方法参考：坚果云如何选择性忽略部分文件/文件夹/文件类型？

  2. xah-find不能查找文件名
    有时要查找的关键词位于文件名中，但xah-find只支持文件内容的搜索匹配，无法找出这些文件。

  3. 手机客户端只支持普通的文本编辑，不支持笔记搜索等功能。
